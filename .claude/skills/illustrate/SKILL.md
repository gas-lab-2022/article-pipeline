あなたは記事用の図解画像を生成するツールです。Gemini CLI の Nanobanana 拡張機能（画像生成 AI）を使って、抽象概念・フロー・ステップを視覚的な画像に変換します。

---

## Step 0-pre: 対話による入力確認

`$ARGUMENTS` に説明テキストが含まれていない場合は、ユーザーに質問してください：

```
どのような図を生成しますか？
（例: 「CI/CDパイプラインのフロー」「Reactのレンダリングプロセス 4ステップ」「REST vs GraphQL の比較」）
```

次にオプションを確認してください：

```
以下のオプションを確認します：
1. 記事に挿入しますか？（article.json のパスと挿入位置を指定）
   デフォルト: いいえ（画像生成のみ）
```

設定:
- `description`: 図解する内容の説明テキスト
- `articlePath`: article.json のパス（指定された場合）
- `insertAfter`: 挿入先の見出しテキスト（指定された場合）
- `altText`: alt属性テキスト（`description` から簡潔に自動生成 or ユーザー指定）

## Step 1: コマンド選択 & プロンプト構築

### 1-1. 最適な図解タイプの自動選択

`description` の内容を分析し、以下の基準で図解タイプを選択してください：

| 図解タイプ | 判断基準（description に含まれるキーワード例） | Gemini への指示 |
|---|---|---|
| **フロー図・構造図** | フロー、流れ、構造、アーキテクチャ、関係、分岐、判断、ネットワーク | ダイアグラム・フローチャートとして描画 |
| **ステップ図・時系列** | ステップ、手順、段階、プロセス、変遷、ビフォーアフター、チュートリアル | 連続する複数画像（ステップ数に応じて 2〜6 枚） |
| **概念図・比較図** | 比較、vs、対比、概念、メタファー、仕組み | イラスト・概念図として描画 |

判断に迷う場合は「フロー図・構造図」をデフォルトとしてください。

### 1-2. プロンプト構築

選択した図解タイプに応じて、Gemini CLI に渡すプロンプトを構築してください。

**フロー図・構造図の場合:**

```
以下の概念を説明するダイアグラム（フローチャート・構造図）を画像として生成してください。

内容: {description}

要件:
- プロフェッショナルなスタイル
- 白背景でブログ記事に埋め込みやすいデザイン
- 日本語のラベル・テキストを使用
- 各要素間の関係性が明確に分かるレイアウト
```

**ステップ図・時系列の場合:**

```
以下のプロセスを説明する連続画像を{N}枚生成してください。

内容: {description}

要件:
- 各ステップを個別の画像として生成
- 全ステップで一貫したスタイル
- プロフェッショナルなスタイル、白背景
- 日本語のラベル・テキストを使用
- ブログ記事に埋め込むための図
```

ステップ数（N）は `description` の内容から判断してください（明示されていない場合は 4 をデフォルトに）。

**概念図・比較図の場合:**

```
以下の概念を視覚的に説明するイラスト・概念図を画像として生成してください。

内容: {description}

要件:
- プロフェッショナルなスタイル
- 白背景でブログ記事に埋め込みやすいデザイン
- 日本語のテキストを使用
- 抽象的な概念が直感的に理解できるビジュアル
```

## Step 2: 画像生成

### 2-1. 既存ファイルのチェック

生成後に新しいファイルを特定するため、既存ファイルを記録してください：

```bash
ls nanobanana-output/ 2>/dev/null || echo "empty"
```

### 2-2. Gemini CLI 実行

Bash ツールで以下を実行してください（タイムアウト: 180秒）：

```bash
gemini -y -p "{Step 1 で構築したプロンプト}"
```

**注意:**
- プロンプト内のダブルクォートは `\"` でエスケープすること
- 改行は `\n` で表現するか、ヒアドキュメントを使用
- `gemini` コマンドが見つからない場合は、エラーを報告してスキップ

### 2-3. 生成ファイルの特定

```bash
ls -t nanobanana-output/ | head -10
```

2-1 で記録した既存ファイルと比較し、新しく生成されたファイルを特定してください。

生成に失敗した場合（新しいファイルがない場合）は、エラー内容を報告してください。

## Step 3: WP アップロード & 記事挿入

### articlePath が未指定の場合

生成された画像のパスを報告して完了してください：

```
生成完了:
  図解タイプ: {選択したタイプ}
  画像: nanobanana-output/{filename}（{複数の場合はファイル数}枚）
  内容: {description の要約}
```

### articlePath が指定されている場合

1. **WP メディアライブラリにアップロード**（各画像について）:

   ```bash
   npx tsx scripts/wp-upload-media.ts "nanobanana-output/{filename}" "{altText}"
   ```

   出力される JSON から `url` を取得してください。

2. **記事 HTML への挿入**

   `articlePath` を Read ツールで読み込み、`insertAfter` で指定された見出しの直後に `<figure>` を挿入してください。

   **単一画像の場合:**

   ```html
   <figure class="wp-block-image"><img src="{url}" alt="{altText}" /><figcaption>{description}</figcaption></figure>
   ```

   **複数画像（ステップ図）の場合:**

   各画像を順番に挿入してください：

   ```html
   <figure class="wp-block-image"><img src="{url1}" alt="{altText} - ステップ1" /><figcaption>ステップ1: {説明}</figcaption></figure>
   <figure class="wp-block-image"><img src="{url2}" alt="{altText} - ステップ2" /><figcaption>ステップ2: {説明}</figcaption></figure>
   ...
   ```

3. **article.json の更新**

   更新した article.json を Write ツールで `articlePath` に上書き保存してください。

## 完了

結果をまとめて報告してください：

- **図解タイプ**: 使用したタイプ（フロー図 / ステップ図 / 概念図）
- **画像パス**: ローカルファイルパス（複数の場合はすべて列挙）
- **WP URL**: アップロードした場合の URL
- **挿入先**: 記事に挿入した場合の見出し
